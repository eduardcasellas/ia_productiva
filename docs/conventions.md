# Conventions

## Purpose

This document defines the conventions used throughout IA-Productiva.

Consistency has priority over personal preference.

---

# Language

Documentation may be written in:

- Catalan
- English

Each file should use a single language consistently.

---

# File Names

Rules:

- lowercase
- hyphens
- descriptive
- no spaces

Correct:

```
project-brief.md
```

Incorrect:

```
Project Brief.md
```

---

# Folder Names

Always lowercase.

Example:

```
content/
resources/
automation/
```

---

# Markdown

Use:

- H1 only once
- H2 for sections
- H3 for subsections

Avoid unnecessary heading levels.

---

# Lists

Prefer short bullet lists.

Avoid deeply nested structures.

---

# Tables

Use only when comparison improves readability.

---

# Code Blocks

Always specify language.

Example:

````text
```bash
git status
```
